# Week 15 — Introduction to Power BI and Power Query Editor

This week I started the **Power BI** module, Microsoft's Business Intelligence tool. I focused on understanding the Power BI ecosystem and on data acquisition and preparation using the **Power Query Editor**.

I learned what distinguishes a report from a dashboard, the two data storage modes (Import vs DirectQuery), and how to profile, clean and transform data before loading it into the model.

## Skills developed:

- Understanding Power BI components (Desktop, Service, Mobile)
- Differentiating between reports, dashboards and visualizations
- Choosing between Import mode and DirectQuery mode
- Connecting data sources (Excel, CSV, Database, Folder)
- Profiling data with Column Quality, Distribution and Profile
- Cleaning data: removing columns, filtering rows, replacing values
- Combining tables with Append and Merge
- Assigning correct data types

## Challenges overcome:

- Understanding when to use Import vs DirectQuery based on requirements
- Identifying key fields through profiling (distinct vs unique)
- Handling inconsistent values with Replace Values
- Optimizing data volume by removing unnecessary columns
- Enabling profiling on entire dataset (not just first 1000 rows)
